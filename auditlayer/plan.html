Executive Summary

What we're building, for whom, and why now.

AuditLayer is a subscription platform that generates deep, structured social media audit reports for brands, creators, and agencies. A client enters a social media handle, answers a few guiding questions, and receives a polished, actionable 15-section audit — competitive metrics, content gaps, audience psychology, growth strategy, visual branding blueprint — delivered as a self-contained HTML report with an optional chat interface for iterative refinement.

The engine is Hermes Agent running the social-media-audit skill — a battle-tested pipeline already proven with real clients, most recently a comprehensive Instagram audit for Dr. Hemal Patel (@hemalpatelphd, 10.7K followers) that generated immediate commercial interest from Narin's network of med-tech marketing teams.

Core insight: The audit pipeline already works. The skill already exists. The report format has been validated by real clients. The gap is not technology — it's productization: a portal, a payment flow, and self-serve access.

Why This Architecture

  • Zero new infrastructure. Everything runs on the existing CX22. No new VMs, no Docker, no databases beyond what Hermes already uses.
  • Cloudflare Tunnel means no open inbound ports. The CX22 makes an outbound connection to Cloudflare. TLS termination, DDoS protection, and OAuth gating all happen at Cloudflare's edge — free for <50 users.
  • Hermes sessions are already isolated. Each audit runs in its own session. The gateway handles concurrency natively. No per-user containers needed for the pilot.
  • The portal is thin. It's a UI shell. All intelligence lives in Hermes + the audit skill. The portal just renders HTML reports and proxies chat messages.

Report Sections (Existing, Proven)

  1. Executive Summary — 2–3 paragraphs, core insight callout
  2. Key Metrics — metric cards + benchmark comparison table
  3. Strengths — 8–10 items, what's working
  4. Weaknesses — 8–10 items, what's broken
  5. Growth Bottlenecks — severity-ranked table with impact assessment
  6. Content Gaps — formats/topics entirely absent
  7. Audience Psychology — primary + secondary audience motivations
  8. Viral Opportunities — high-leverage content ideas
  9. Monetization Opportunities — revenue levers from social presence
  10. Performance Score — 8-dimension weighted score out of 100
  11. Competitive Context — competitor comparison table
  12. High-Impact Recommendations — 3 tiers (★★★★★ to ★★)
  13. Platform-Specific Improvements — per-platform tactical guidance
  14. Content Ideas — 10 specific content pieces to produce
  15. 30-Day Optimization Strategy — week-by-week plan with success metrics

Key advantage: This skill improves over time. Every audit run refines the prompts, data sources, and analysis patterns. After 50 audits, the skill is materially better than a fresh install — and that accumulated quality is impossible for a competitor to replicate.

Guided Questions (New — Replaces Raw Prompt)

Instead of exposing a blank chat or raw configuration, the portal asks 3–5 guiding questions that shape the audit:

  1. "What's the primary goal for this audit?" → Growth strategy / Competitive positioning / Content optimization / Rebrand assessment
  2. "Any specific competitors to benchmark against?" → Free-text, optional
  3. "Anything unusual about this account we should know?" → Recent pivot, controversy, platform migration
  4. "Preferred depth?" → Standard (15 sections) / Deep (25+ sections, premium tier)
  5. "Target audience context?" → B2B / B2C / Creator-personal-brand

These answers are injected into the audit skill's prompt as context parameters — the client never touches a system prompt.

Token Budget & Quality Guardrails

Each audit run is configured with sensible limits that preserve quality while preventing runaway costs:

  • Max agent iterations: 30 (typically uses 12–18 for a full audit)
  • Max web searches: 15 per audit
  • Per-section budget: ~2,000 tokens for analysis (sections auto-pruned if budget exceeded)
  • Model tiering: DeepSeek v4 for analysis & writing; cheaper flash model for data extraction & formatting
  • Hard cap: $3.00 per audit — if exceeded, the report is delivered as-is with a note

These limits are configured in the API proxy, not exposed to clients. They can be relaxed for Enterprise tier or "Deep Audit" add-ons.

Implementation Roadmap

Four phases. Each delivers standalone value.

Phase 1 · Weeks 1–2

Pilot Portal + Manual Delivery

Build the thin portal (Next.js, ~500 LOC): login via Clerk/Cloudflare Access, "New Audit" form with guided questions, report viewer (renders existing HTML format), download button. Cloudflare Tunnel for public HTTPS. Hermes generates reports manually triggered (Ashesh or Narin approves). Stripe Payment Links for billing (no integration needed yet). Deliver to Narin's first 3–5 clients.

Deliverable: Portal live at a domain. First paying users.

Phase 2 · Weeks 3–4

Self-Serve Generation

Build the API proxy (Express, ~200 LOC): receives audit requests from the portal, queues them, calls Hermes API, tracks cost per audit, returns report URL. Add email notification ("Your audit is ready"). Stripe webhook integration: subscriptions auto-enable/disable accounts. Remove manual trigger requirement.

Deliverable: Client hits "Generate" → gets report without human in the loop.

Phase 3 · Weeks 5–6

Chat Refinement + Quality Polish

Add the in-report chat panel: client highlights a section and types a refinement. Chat messages route through the API proxy to the same Hermes session. Add "Compare" feature (Pro tier): side-by-side audit of two accounts. Skill tuning sprint with Narin: run 10 test audits on diverse account types, refine prompts until quality is consistently high without manual review.

Deliverable: Chat refinement works. Compare feature live. Skill quality is vetted.

Phase 4 · Weeks 7+

Scale + Platform Expansion

Add TikTok and YouTube audit support (reuse the same skill framework; different data sources). Multi-platform audits (single report covering Instagram + TikTok). PDF export (Hermes generates LaTeX → pdflatex; texlive already installed). White-label for Enterprise tier (client's logo on reports). If concurrency becomes a bottleneck, add a second CX22 node behind a load balancer.

Deliverable: Multi-platform. PDF. White-label. Ready for 50+ users.

Risk Analysis

What could break, and how we handle it.

RiskSeverityLikelihoodMitigation
Instagram API / scraping restrictions High Medium Diversify data sources from day 1 (TikTok API, YouTube API, Social Blade). Never depend on a single data pipeline. Flag reports with incomplete data transparently.
Report quality inconsistency High Medium Version-lock skill prompts. Run quality checks post-generation. Offer "regenerate" if client rates <3/5. Track quality scores per audit; intervene if trend declines.
Narin becomes the bottleneck Medium High Phase 2 eliminates manual trigger. Phase 3 eliminates manual review. First 10 audits per client get Narin's eyes; after that, automated with quality monitoring.
Cost overruns per audit Medium Medium Hard cap of $3.00/audit at the proxy level. Per-section token budgets. Model tiering (cheap for extraction, premium for analysis). Alert if any audit exceeds $2.00.
Concurrency bottleneck on CX22 Low Low Hermes sessions are independent. CX22 handles ~10 concurrent audits fine. For Phase 4 (50+ concurrent), add a second node. Queue + email notification means users don't wait synchronously.
Prompt injection / skill leakage Low Medium Chat refinement is scoped to the report session, not the system prompt. The API proxy strips anything that looks like configuration leakage before returning to the client.
"Same report, worse this month" Medium Medium Version-locked prompts. Offer "compare to previous" as a feature — turns a bug into a product capability. Track audit quality with post-delivery 1–5 rating.

Competitive Moat

Why this is defensible, not just another LLM wrapper.

Moat 1

Self-Improving Skill

Every audit makes the skill better. After 50 audits, it's materially superior to a fresh install. A competitor starting from scratch gets the generic version — and takes 50 audits to catch up to where we are today.

Moat 2

Domain Calibration

Narin knows what a "good" audit looks like for med-tech/wellness/creator audiences. The skill encodes that. Generic competitors don't know that a UCSD professor's audit should prioritize credibility signals over engagement hacks.

Moat 3

Structured Consistency

The 15-section framework produces consistent, comparable output. Clients know what they're getting. Generic LLM audits vary wildly in structure and depth. Consistency builds trust; trust builds retention.

Moat 4

Sticky Refinement History

When a client iterates on a report through chat, those refinements live in their session history. Switching providers means starting over — and losing the accumulated context their agent has about their brand.

Why "just use ChatGPT for this" doesn't work: ChatGPT cannot run a 15-section audit pipeline with web search, Social Blade data extraction, benchmark comparison, competitive analysis, and structured HTML generation in a single pass. It can write a social media tip. It cannot produce an @hemalpatelphd-level report. The skill's accumulated structure, scoring framework, and domain calibration are the product — not the LLM underneath.
